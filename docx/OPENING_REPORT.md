## 1. 课题背景与研究意义

随着制造业数字化与智能化水平不断提高，工业设备的运行可靠性已经成为生产效率、产品质量和维护成本控制的关键因素。传统设备维护方式主要包括事后维修和定期维护两类。

事后维修的特点是故障发生后再进行处理，这种方式实现简单，但往往会造成以下问题：

1. 设备停机具有突发性，容易带来较大生产损失。
2. 故障传播可能导致关联部件损伤，增加维修成本。
3. 维修时间不可控，不利于生产计划安排。

定期维护虽然能够一定程度降低突发故障风险，但同样存在明显不足：

1. 维护周期通常依据经验给出，缺乏设备个体差异考虑。
2. 设备未真正退化到危险阶段时就被维护，存在过度维护问题。
3. 备件和人工投入较高，整体维护经济性不佳。

因此，工业界逐渐从“被动维修”转向“预测性维护（PdM）”。预测性维护的核心思想是在设备完全失效之前，通过监测历史数据和当前状态，对其健康状态、故障风险和剩余寿命进行预测，从而实现更合理的维护决策。

在大量旋转机械中，滚动轴承属于故障高发且影响范围广的重要部件。轴承退化通常伴随振动模式变化、局部冲击增强、频域能量偏移和温度上升等现象。因此，面向轴承构建故障预测系统，既具有直接工程意义，也适合作为数据驱动预测性维护的典型研究对象。

本课题的意义主要体现在以下几个方面：

1. 工程意义：为设备寿命预测和维护预警提供实验基础。
2. 教学意义：覆盖需求分析、系统设计、实现、测试和文档等软件工程完整链路。
3. 研究意义：探索多模型、多指标、多数据源下的轴承退化建模流程。

## 2. 国内外研究与现状分析

轴承故障预测的研究大致可以分为三类方法：

### 2.1 基于人工经验与统计特征的方法

早期方法主要依赖时域和频域特征，例如均值、均方根、峰值、峭度、频谱能量、主频等，再结合传统机器学习方法完成健康识别或寿命回归。这类方法可解释性较强、实现成本较低，但通常对特征设计依赖较重，泛化能力受限。

### 2.2 基于深度学习的端到端方法

随着深度学习的发展，越来越多工作直接使用振动信号窗口作为模型输入，通过 CNN、RNN、LSTM、Transformer 等模型自动学习特征表达。其优点是减少了手工特征依赖，能够在复杂退化模式下获得更强拟合能力，但也带来训练成本更高、实验管理复杂、可解释性不足等问题。

### 2.3 面向工程落地的综合实验平台

在实际工业场景中，仅有模型还不够，还需要：

1. 统一的数据接入能力。
2. 可配置的预处理流程。
3. 训练过程监控与报警能力。
4. 实验配置与结果的自动记录能力。
5. 可用于答辩和报告的可视化输出能力。

从课程项目视角看，现有很多实现停留在“脚本 + 单模型 + 单次实验”的水平，缺少系统化设计，也难以支撑软件工程课程的完整交付。因此，本课题拟构建一个以工程框架为导向的轴承故障预测系统。

## 3. 项目目标

### 3.1 总体目标

开发一个面向工业轴承故障预测的实验系统，实现从数据加载、信号处理、模型训练到结果可视化与实验记录的完整流程。

### 3.2 具体目标

1. 支持 XJTU-SY 和 PHM2012 两个真实数据集接入。
2. 支持振动信号与温度信息的统一抽象。
3. 支持滑动窗口、归一化、鲁棒裁剪等预处理方法。
4. 支持时域与频域特征提取。
5. 支持 3σ 与 FPT 两种阶段划分方法。
6. 支持 CNN、RNN、Transformer、MLP 等多种建模方式。
7. 支持端到端预测、滚动预测和不确定性建模。
8. 支持 EarlyStopping、TensorBoard 和梯度异常报警。
9. 支持实验配置、训练历史、指标与预测结果自动记录。
10. 支持混淆矩阵、退化阶段图、预测曲线、注意力热图等结果展示。

### 3.3 预期成果

1. 一套可运行的工业轴承故障预测系统。
2. 真实数据上的加载、训练与评估验证结果。
3. 完整课程文档：开题报告、SRS、SDD、UML。
4. 自动化测试与实验复现材料。

## 4. 研究内容与关键问题

### 4.1 研究内容

本项目主要研究和实现以下内容：

1. 真实轴承退化数据的标准化接入。
2. 原始振动信号的预处理与窗口化。
3. 基于时域和频域的特征工程。
4. 面向 RUL、阶段分类和滚动预测的任务构造方法。
5. 多种模型下的统一训练与测试框架。
6. 训练过程监控、异常报警与实验追踪。
7. 面向课程答辩的结果展示和文档输出。

### 4.2 关键技术问题

#### 4.2.1 多数据集格式差异问题

不同数据集的目录层次、文件命名和字段结构并不一致。例如：

1. XJTU-SY 主要是两列振动数据，部分文件带表头。
2. PHM2012 同时包含加速度与温度文件，且目录分组更复杂。

因此，需要一个统一的实体抽象，将不同来源数据转化为一致的数据对象。

#### 4.2.2 原始信号到模型输入的映射问题

轴承原始信号通常是长序列，无法直接作为训练样本使用。需要解决：

1. 如何切窗。
2. 如何归一化。
3. 如何提取稳定且有效的特征。
4. 如何让不同任务共享这一套流程。

#### 4.2.3 多任务统一框架问题

RUL 预测、退化阶段分类和健康指标滚动预测虽然目标不同，但又共享相当多的底层能力。如何设计统一框架，使其既能复用基础模块，又不导致接口混乱，是本项目的关键工程问题。

#### 4.2.4 训练过程可观测性问题

很多课程项目能够“跑起来”，但训练过程几乎不可观测。为了提高工程完整性，需要解决：

1. 配置自动记录。
2. TensorBoard 可视化。
3. 梯度消失与爆炸报警。
4. EarlyStopping 控制。

#### 4.2.5 结果可解释与可展示问题

仅输出数字指标难以支撑答辩展示，因此系统必须能够产生：

1. 预测曲线。
2. 退化阶段图。
3. 混淆矩阵。
4. 注意力热图。

## 5. 技术路线

本项目采用“真实数据接入 + 统一实体抽象 + 任务化标注 + 模型训练 + 实验记录 + 可视化展示”的技术路线。

```text
真实数据集 / 合成数据
        ↓
数据加载器（XJTULoader / PHM2012Loader）
        ↓
BearingEntity 统一抽象
        ↓
预处理（裁剪 / 归一化 / 切窗）
        ↓
特征提取 / 健康指标构造
        ↓
任务标注（RUL / Stage / HI Forecast）
        ↓
模型训练（CNN / RNN / Transformer / MLP）
        ↓
测试与预测（Direct / Rolling / MC Dropout）
        ↓
评估（MAE / RMSE / PHM2012Score / NASA Score）
        ↓
可视化与实验记录
```

## 9. 预期风险与应对措施

| 风险 | 描述 | 应对措施 |
| --- | --- | --- |
| 数据下载风险 | 官方镜像可能不稳定 | 准备多个镜像源，保留本地缓存 |
| 数据格式差异风险 | 不同数据集文件结构不一致 | 使用统一加载器抽象和真实数据测试 |
| 训练结果不稳定 | 模型对超参数敏感 | 缩小 smoke test 范围，逐步调参 |
| 工程复杂度上升 | 多模块协作可能导致耦合 | 采用分层设计和回调机制 |
| 文档滞后 | 实现完成后再补文档容易遗漏 | 与实现同步维护文档和测试 |
